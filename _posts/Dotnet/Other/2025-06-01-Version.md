# 语义化版本控制

- **主版本**：最左边的数字。例如 `1.0.0` 中的 `1`。  
  此数字发生更改意味着代码可能出现**破坏性变更**，可能需要重写部分代码。

- **次要版本**：中间的数字。例如 `1.2.0` 中的 `2`。  
  此数字发生更改意味着**新增了向后兼容的功能**。你的现有代码仍可正常工作，接受更新通常是安全的。

- **修补程序版本**：最右边的数字。例如 `1.2.3` 中的 `3`。  
  此数字发生更改意味着**修复了 bug 或做了微小改进**，不影响公共 API。接受更新应是安全的。

## 版本号变更示例

| 类型         | 变更示例               |
|--------------|------------------------|
| 主版本       | `1.0.0` → `2.0.0`     |
| 次要版本     | `1.1.1` → `1.2.0`     |
| 修补程序版本 | `1.0.1` → `1.0.2`     |

## NuGet 版本范围表示法

不用记

| 表示法        | 匹配规则             | 说明                               |
|---------------|----------------------|------------------------------------|
| `1.0`         | `x >= 1.0`           | 最低版本（包含）                   |
| `(1.0,)`      | `x > 1.0`            | 最低版本（不包含）                 |
| `[1.0]`       | `x == 1.0`           | 精确版本匹配                       |
| `(,1.0]`      | `x <= 1.0`           | 最高版本（包含）                   |
| `(,1.0)`      | `x < 1.0`            | 最高版本（不包含）                 |
| `[1.0,2.0]`   | `1.0 <= x <= 2.0`    | 闭区间（两端包含）                 |
| `(1.0,2.0)`   | `1.0 < x < 2.0`      | 开区间（两端不包含）               |
| `[1.0,2.0)`   | `1.0 <= x < 2.0`     | 左闭右开区间                       |
| `(1.0)`       | —                    | 无效                               |

## 在 `.csproj` 中使用版本范围示例

```xml
<!-- 接受 6.1 及更高版本 -->
<PackageReference Include="ExamplePackage" Version="6.1" />

<!-- 接受任意 6.x.y 版本 -->
<PackageReference Include="ExamplePackage" Version="6.*" />
<PackageReference Include="ExamplePackage" Version="[6,7)" />

<!-- 接受高于 4.1.3 的任意版本（用于确保包含某个 bug 修复） -->
<PackageReference Include="ExamplePackage" Version="(4.1.3,)" />

<!-- 接受低于 5.0 的任意版本（不推荐，因难以确定最早兼容版本） -->
<PackageReference Include="ExamplePackage" Version="(,5.0)" />

<!-- 接受 1.x 或 2.x 版本，但不包括 0.x 和 3.x+ -->
<PackageReference Include="ExamplePackage" Version="[1,3)" />

<!-- 接受从 1.3.2 到 1.4.x 的版本，但不包括 1.5+ -->
<PackageReference Include="ExamplePackage" Version="[1.3.2,1.5)" />
