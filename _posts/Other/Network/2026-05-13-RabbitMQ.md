# RabbitMQ

需要安装，和数据库差不多的后台程序

要先安装`erlang`语言,必需依赖.

再安装`RabbitMq`

这里用到`erlang 27.x` `rabbitMQ 4.3.0`

## .Net

`RabbitMQ.Client` 包

默认端口为`5672`

发布者和消费者通用代码
```C#
//创建工厂
ConnectionFactory factory = new ConnectionFactory { HostName = "localhost" };
//创建连接
using IConnection connection = await factory.CreateConnectionAsync();
//创建管道
using IChannel channel = await connection.CreateChannelAsync();

//声明队列，幂等
await channel.QueueDeclareAsync(
    //队列名
    queue: "task_queue",
    //是否持久化
    durable: true,
    //独有
    exclusive: false,
    //自动删除
    autoDelete: false,
    //
    arguments: new Dictionary<string, object?> { { "x-queue-type", "quorum" } }
);
```

发布者
```C#
//发布任务,body为具体信息,exchange为交换模式，空字符串为默认.
await channel.BasicPublishAsync(exchange: string.Empty, routingKey: "task_queue", body: body);
```

消费者
```C#
//设置消费者Quality of Service ,prefetchCount是可以接收的数量
await channel.BasicQosAsync(prefetchSize: 0, prefetchCount: 1, global: false);

//创建消费者
AsyncEventingBasicConsumer consumer = new AsyncEventingBasicConsumer(channel);

//添加消费者的接收动作，ea是发送过来的信息类
consumer.ReceivedAsync += async (model, ea) =>
{
    //body是byte[]
    var body = ea.Body.ToArray();
    string message = Encoding.UTF8.GetString(body);

    //主动确认完成delivertTag基本是唯一标识
    await channel.BasicAckAsync(deliveryTag: ea.DeliveryTag, multiple: false);
};

//消费任务
await channel.BasicConsumeAsync("task_queue", autoAck: false, consumer: consumer);
```
### 交换模式

type:
*   "direct"：精确匹配 routing key
*   "fanout"：广播到所有绑定队列
*   "topic"：模式匹配 routing key
*   "headers"：基于消息 headers 匹配(较少用)  

>[!warning]
>默认的交换模式是特殊的`direct`，无法实现广播.

```C#
/*await channel.ExchangeDeclareAsync(
    exchange: "logs",
    type: "fanout",
    durable: true,
    autoDelete: false,
    arguments: null
);*/

//扇形模式
await channel.ExchangeDeclareAsync(
    //自定义名称
    exchange: "logs", 
    //类型
    type: ExchangeType.Fanout
    );

//发布的时候使用扇形类型,Fanout的routingKey为空.
await channel.BasicPublishAsync(exchange: "logs", routingKey: string.Empty, body: body);
```

**topic 模式,能够操作字符**

被传入的routingKey必须是单词列表.由`.`划分，例如`quick.orange.rabbit`

`*` 可以代替一个单词

`#` 可以代替零或者多个单词

绑定密钥的时候`*.orange.*` `*.*.rabbit` `quick.#`

### 广播模式

使用Fanout模式，不需要持久，用完既丢。
```C#
//不加参数，非持久，仅当前连接可用，自动删除
QueueDeclareOk queueDeclareResult = await channel.QueueDeclareAsync();
//名字为自动生成的
string queueName = queueDeclareResult.QueueName;

//消费者使用，因为临时队列没有设置exchange
await channel.QueueBindAsync(queue: queueName, exchange: "logs", routingKey: string.Empty);
```

> **除了Fanout模式,direct也可以多播.**

如果使用了默认队列

发布者`routingKey`必须要和队列的名字相同，但是同名又只能存在一个。所以无法多播。(默认队列的名字和routingKey是同一个)
```C#
await channel.QueueDeclareAsync(
    queue: "yourQueue",
    durable: true,
    exclusive: false,
    autoDelete: false,
    arguments: new Dictionary<string, object?> { { "x-queue-type", "quorum" } }
);

//routeingKey必须和queue:相同.但是如果相同那么同名只能有一个队列，不能多播，只能排队得到
await channel.BasicPublishAsync(exchange: string.Empty, routingKey: "task_queue", body: body);
```

direct多播

发布者按照routingKey发布给所有绑定了这个routingkey的队列.每个队列都能得到一份.

匿名队列的名字可以和routingKey不同.

```C#
//创建交换
await channel.ExchangeDeclareAsync("direct_logs", ExchangeType.Direct);

//发布者
await channel.BasicPublishAsync(exchange: "direct_logs", routingKey: "warn", body: bytes);

//消费者
//创建匿名队列,注意匿名队列是临时的,如果发布者在消费者存在之前发布，消息会消逝.
QueueDeclareOk queueDeclareOk = await channel.QueueDeclareAsync();
string queueName = queueDeclareOk.QueueName;

//将队列绑定到两个routingKey,这样他就能接收到两个routing的信息
await channel.QueueBindAsync(queueName, "direct_logs", "warn");
await channel.QueueBindAsync(queueName, "direct_logs", "info");
```


## UI可视化管理

`C:\Program Files\RabbitMQ Server\rabbitmq_server-4.3.0\sbin` 在该地址使用命令行

`rabbitmq-plugins enable rabbitmq_management` 启动UI管理器

`localhost:15672` 默认地址端口 

`guest` 默认用户名和密码

`rabbitmqctl` 这个命令可以管理用户权限
