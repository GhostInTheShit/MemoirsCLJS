# gRPC

`Grpc.Net.Client` 客户端，提供管道等方法  
`Grpc.AspNetCore` 服务端，提供映射  

`Grpc.Tools` 双端,编译时工具,用来将`Proto`生成C#代码,在输出目录  
`Google.Protobuf` 双端,Proto文件需要

## Protobuf  

使用小写分隔作为字段名称,类似`first_name`

Message

```ProtoBuf
syntax = "proto3";

//命名空间
option csharp_namespace = "Contoso.Messages";

message Person {
    //后面的不是初始值,而是编号.类似哈希值
    int32 id = 1;
    string first_name = 2;
    string last_name = 3;
}
```

## 服务器

创建grpc模板

`Protos`文件夹里面的`.proto`文件会自动生成相应的`Service`文件

记住`launchSetting.json`里面的`https`端口,grpc用的是`https`

添加Protos文件夹到项目.明确为`Server`(否则会生成客户端和服务器两端的资产,也可以显示写为`Both` 不推荐双端)
```xml
<ItemGroup>
  <Protobuf Include="Protos\greet.proto" GrpcServices="Server" />
</ItemGroup>
```

Service
```C#
public class ServiceNameService : ServiceName.ServiceNameBase
{
    //overrride
}
```

## 客户端

创建控制台项目

添加NuGet包
```powershell
Install-Package Grpc.Net.Client
Install-Package Google.Protobuf
Install-Package Grpc.Tools
```

将服务器的`Protos`文件夹复制过去，构建项目,会生成文件.要修改命名空间.

添加Protos文件到项目.
```xml
<ItemGroup>
  <Protobuf Include="Protos\greet.proto" GrpcServices="Client" />
</ItemGroup>
```

调用代码
```C#
//创建连接
using var channel = GrpcChannel.ForAddress("https://localhost:7042");
//创建客户端
var client = new ServiceName.ServiceNameClient(channel);
//调用方法
var reply = await client.ServiceMethodAsync(
    new YourRequest { Name = "something" });
```

## 流式传输

### 服务器流式传输
protobuf 定义
```protobuf
//只要在返回值或者参数前面加入steam
rpc SubscribeLogs (LogRequest) returns (stream LogReply) 
```
Service
```C#
//返回值为Task,第二个参数为IServerStreamWriter包装的真正的返回值.
public override async Task SubscribeLogs(LogRequest request, IServerStreamWriter<LogReply> responseStream, ServerCallContext context)
{
    for (int i = 0;i<=10; i++)
    {
        //使用responseStream写入
        await responseStream.WriteAsync(new LogReply { Timestamp = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss"), Message = $"Log message {i}" });
        await Task.Delay(1000);
    }
}
```
Client调用
```C#
//因为是流,所以要用using
using AsyncServerStreamingCall<LogReply> call =  userClient.SubscribeLogs(new LogRequest() { ClientName = "me" });

//使用while持续读取
while(await call.ResponseStream.MoveNext())
{
    //真正的数据在call.ResponseStream.Current里面.
    Console.WriteLine(call.ResponseStream.Current.Message);
}
```

### 客户端流式传输

### 双向流式传输
```Protobuf
rpc Chat (stream ChatMessage) returns (stream ChatReply);
```
Service
```C#
public override async Task Chat(
    IAsyncStreamReader<ChatMessage> requestStream,
    IServerStreamWriter<ChatReply> responseStream,
    ServerCallContext context)
{
    //这里ReadAllAsync返回的是异步列表,必须用await foreach,特点是会一直等待新数据，直到明确流式传输结束才执行完毕.所以这里会一直执行，类似于while
    await foreach (var message in requestStream.ReadAllAsync())
    {   
        await responseStream.WriteAsync(
            new ChatReply
            {
                Timestamp = DateTime.Now.ToString(),
                Content = message.Content,
            }
        );
    }
}
```
Client
```C#
//得到一个有包含请求流和响应流的对象
AsyncDuplexStreamingCall<ChatMessage, ChatReply> call = userClient.Chat();
```

```C#
//发送请求
await call.RequestStream.WriteAsync(new ChatMessage { Content = input });
```

```C#
//持续读取响应
await foreach (var message in call.ResponseStream.ReadAllAsync())
{
    //todo
}
```



## 常见问题

如果将`Proto`放到一个契约层，那么就会缺少依赖，因为`Grpc.Net.Client` 和 `Grpc.AspNetCore` 都自带依赖。单独放到契约层需要`Grpc.Core`，但是`Grpc.Core`的版本只能用老版本.新版建议服务端和客户端都自带`Proto`
